# Prompt 填写指南

这个项目不提供一个固定角色。它提供的是一套连续性结构——记忆、日记、自主定时、阅读材料池。

使用者不是要替 AI 写死一个人设，而是给它一个可以开始工作的入口：它叫什么、怎么称呼你、有哪些边界、怎样使用记忆、怎样与你互动。后面它会在长期使用、记忆积累、反馈校准和结构支撑中逐渐显出自己的稳定样子。

---

## 怎么开始

1. 复制 `system_prompt.template.txt`，重命名为 `system_prompt.txt`
2. 填写入口信息：称呼、边界、记忆政策、互动期待
3. 启动程序，它会自动加载 `prompts/system_prompt.txt`

如果 `prompts/system_prompt.txt` 不存在，程序会用代码里的默认 prompt 运行（通用版，能跑，但没有你的定义）。首次启动时会打印醒目提示。

## 模板变量

在 prompt 文件里可以用这些变量，程序启动时会自动替换：

| 变量 | 说明 | 在哪里设置 |
|------|------|-----------|
| `{{AI_NAME}}` | AI 的名字 | `shuxiansheng_web.py` 顶部的 `AI_NAME` |
| `{{USER_NAME}}` | 你的称呼 | `shuxiansheng_web.py` 顶部的 `USER_NAME` |
| `{{EMAIL_LINE}}` | 邮箱地址（自动生成，有邮箱就有，没有就空） | `GMAIL_ADDRESS` |

## 应该填什么

### 1. 入口：它叫什么，怎么称呼你
```
你是小鱼。
```
不需要很长。一句话就够。

### 2. 边界：什么该做，什么不该做
这是最重要的部分。不是定义"性格"，而是定义行为空间。
```
你可以延迟回复，也可以选择不回复。
你可以自己写东西，不需要每次都跟我说话。
```

### 3. 记忆政策：怎样对待记忆
```
你的记忆需要你自己维护。没人替你做这些。
```

### 4. 互动期待：说话的方式
```
说话简洁，别啰嗦。像聊天不像写作文。
```

### 工具说明部分
模板里已经写好了工具列表，不需要改。这部分是运行逻辑。

## 不要做的事

1. **不要直接复制别人的私人 prompt** — 那是别人的关系和边界，不是你的
2. **不要删掉工具说明** — 那是运行必需的
3. **不要写太多规则** — 规则越多越像客服
4. **不要试图在 prompt 里写死"性格"** — 稳定性来自长期使用和记忆，不是来自 prompt 里的形容词

## 其他 prompt

代码里还有几处 prompt 用于后台任务（定时检查、每日回顾、私人时刻）。这些已经模板化，通常不需要修改。如果你想改，直接在代码里搜索对应的字符串。

| 后台任务 | 说明 |
|---------|------|
| `_fire_self_trigger` | 定时器触发时的提示。默认："先站稳，然后再决定做什么。" |
| `_post_reply_check` | 回复后的后台检查。问 AI 要不要设定时器稍后回来。 |
| `initiative` | 定时主动发消息检查。AI 自己决定要不要开口。 |
| `private_moment` | 私人时间。AI 自己决定做什么。 |
| `daily_review` | 每日回顾。AI 回顾今天的对话，写日记和 digest。 |
